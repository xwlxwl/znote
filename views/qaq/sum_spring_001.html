# 综合入门

1.hibernate 和 mybatis 区别？
  1. hibernate自动，mybatis半自动
  2. 日志 hibernate自带
  3. 移植性 hibernate：hql qbc面向对象,更换数据库,只更换方言即可
  4. 优化 mybatis自己写sql灵活方便
  5. 一级缓存
    hibernate底层是Hashtable 线程安全
    mybatis底层是HashMap 线程不安全

23.过滤器和拦截器的区别
  • 过滤器：
    在web.xml中配置,由servlet实例化对象,spring不能注入,什么请求都拦截
  • 拦截器：
    只拦截后台请求, 都是由spring进行管理的,springmvc除了jsp之外都拦截
24.struts2和springmvc的区别
  1. 入口
    struts2：web.xml filter元素,容器启动初始化
    mvc: web.xml servlet元素,第一次触发请求实例化对象
  2. 创建对象
    struts2： 基于类开发,发送请求时,每次都会创建对象,多例的
    mvc: 基于方法开发,一般设计为单例(默认)
  3. 接受请求参数
    struts2: 成员变量 依赖类存在
    mvc: 局部变量 依赖方法存在

26.事务的 隔离机制和七种传播途径

  • 脏读:读到了其他事物未提交的数据

  • 不可重复读:一次读取到记录之后其他事物对这条数据进行了修改,再次读取数据不一致

  • 幻读:相同的查询条件首次查询后,其他事物添加或删除了新的数据,再次查询不一致

  • 隔离机制：(解决脏读,不可重复读,幻读)

脏读 不可重复读 幻读
Read uncommitted : 会 会 会
Read committed : 不会 会 会
Repeatable read : 不会 不会 会
Serializable : 不会 不会 不会
  • 传播行为:
    国内最常用的就是propagation_required 事务的操作有异常是都得向上抛,不能向上抛就自己new一个异常,不要try(非要try的话,就自己在造一个异常)。

31.webService如何使用

  • 服务端：

    1. 先在web.xml文件中配置
    2. 配置服务端cxf.xml文件
    3. 定义一个bean类
    4. 争对这个bean写一个接口,在接口前加注解@javax.jws.WebService
    5. 在这个接口里写响应的增删该查(定义方法的参数和返回值时 注意不要使用Map对象)
    6. 针对接口写实现类
    7. 在实现类上加@javax.jws.WebService ，@org.springframework.stereotype.Service //这是一个服务层 在类里写响应的实现

  • 客户端：

    1. 新建一个测试项目
    2. 配置客户端cxf.xml文件 （具体使用见下一题）
      将生成的文件复制到工程中,新建一个test类测试
